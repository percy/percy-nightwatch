Card 1

This card demonstrates responsive grid behavior. On mobile, cards stack vertically. On tablet, they arrange in 2 columns. On desktop, they form a 3-column layout.

Card 2

The layout automatically adapts to different screen sizes using CSS Grid and media queries. This tests the SDK's ability to capture different layouts at various breakpoints.

Card 3

Percy captures the DOM at multiple widths when responsiveSnapshotCapture is enabled, allowing visual testing across all device sizes in a single test.

Card 4

The SDK supports Chrome DevTools Protocol (CDP) resizing for fast window changes, with automatic fallback to WebDriver APIs for browsers that don't support CDP.

Card 5

Resize synchronization is handled using PercyDOM.waitForResize(), which waits for the resize event counter to update before capturing the DOM snapshot.

Card 6

After capturing all requested widths, the SDK restores the browser window to its original dimensions, ensuring clean test state management.
